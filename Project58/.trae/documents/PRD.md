# 河道周边环境巡查记录系统 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位
本系统是一款面向河道管理部门的环境巡查记录管理平台，旨在实现河道周边环境巡查工作的数字化、规范化管理，提高巡查效率，保障河道环境质量。

### 1.2 目标用户
- 河道巡查人员：录入巡查点位、记录巡查问题
- 整改负责人员：填写整改情况备注
- 管理人员：查看巡查台账汇总、统计分析

### 1.3 核心价值
- 实现巡查工作全流程数字化管理
- 规范巡查记录格式，便于追溯和统计
- 实时掌握河道环境状况，及时发现和整改问题
- 生成完整的巡查台账，为管理决策提供数据支撑

## 2. 功能需求

### 2.1 巡查点位管理
- 新增巡查点位：录入点位名称、位置、描述、所属河道等信息
- 点位列表查询：支持按河道、区域、状态等条件筛选
- 点位信息编辑：修改点位基本信息
- 点位删除：删除不再使用的巡查点位

### 2.2 巡查问题记录
- 新增巡查记录：选择点位、记录巡查时间、问题类型、问题描述、现场照片等
- 记录列表查询：支持按点位、时间范围、问题类型、整改状态筛选
- 记录详情查看：查看完整的巡查记录信息
- 记录编辑与删除：修改或删除已录入的巡查记录

### 2.3 整改情况备注
- 录入整改信息：针对巡查问题记录整改措施、整改时间、整改责任人、整改后照片
- 整改进度跟踪：标记整改状态（未整改、整改中、已完成）
- 整改历史查看：查看问题的完整整改过程

### 2.4 巡查台账汇总
- 按时间范围统计：月度、季度、年度巡查统计
- 按点位统计：各点位巡查频次、问题数量统计
- 按问题类型统计：各类问题的发生数量和整改率
- 整改情况统计：未整改、整改中、已完成问题数量
- 数据导出：支持导出巡查台账报表

## 3. 非功能需求

### 3.1 性能要求
- 系统响应时间：页面加载 < 2秒，API响应 < 500ms
- 支持并发用户数：≥ 50人同时在线
- 数据查询性能：万级数据查询 < 1秒

### 3.2 安全要求
- 数据持久化存储，定期备份
- 输入数据合法性校验，防止SQL注入
- 操作日志记录，便于审计追溯

### 3.3 兼容性要求
- 支持主流浏览器：Chrome、Firefox、Edge、Safari
- 响应式设计，支持PC端和移动端访问

### 3.4 语言要求
- 系统界面全部使用中文显示

## 4. 用户界面设计

### 4.1 设计风格
- 采用政务/环保系统风格，主色调为蓝色系，代表清洁、专业
- 简洁大方，操作便捷，注重实用性
- 数据可视化展示，直观清晰

### 4.2 页面结构
- 顶部导航栏：系统名称、主要功能菜单
- 左侧菜单栏：各功能模块快捷入口
- 主内容区：当前功能页面内容
- 底部信息栏：版权信息、系统版本

### 4.3 核心页面
1. 首页/仪表盘：数据概览、统计图表
2. 巡查点位管理页：点位列表、新增/编辑表单
3. 巡查记录管理页：记录列表、详情查看、新增表单
4. 整改管理页：待整改列表、整改信息录入
5. 台账汇总页：统计图表、数据筛选、导出功能

## 5. 数据模型

### 5.1 巡查点位（InspectionPoint）
- ID、点位名称、位置描述、所属河道、经度、纬度、区域、状态、创建时间、更新时间

### 5.2 巡查记录（InspectionRecord）
- ID、点位ID、巡查时间、巡查人员、问题类型、问题描述、严重程度、现场照片、状态、创建时间、更新时间

### 5.3 整改记录（RectificationRecord）
- ID、巡查记录ID、整改措施、整改时间、整改责任人、整改后照片、整改状态、备注、创建时间、更新时间

## 6. 技术实现要点

### 6.1 技术栈
- 后端：Go + Fiber 框架
- 数据库：MySQL 8.0
- 前端：HTML + CSS + JavaScript（原生或轻量框架）
- 容器化：Docker + Docker Compose

### 6.2 部署要求
- 提供 Docker 部署方案
- 提供一键启动脚本
- 系统启动时自动初始化数据库和示例数据
